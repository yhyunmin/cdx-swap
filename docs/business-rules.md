# 제품 규칙

프로필은 계정 그 자체가 아니라 Codex auth 파일의 저장 스냅샷이다. 같은 이메일이라도 서로 다른 프로필 이름이면 별도 스냅샷으로 취급한다. 프로필 이름은 파일 경로 일부가 되므로 빈 문자열이나 위험한 경로 문자를 허용하면 안 된다.

현재 Windows Codex 계정은 `%USERPROFILE%\.codex\auth.json`에 있는 기본 auth 파일로 결정된다. 저장된 프로필 중 하나와 파일 내용이 같거나 안정적인 계정 식별자가 같으면 등록된 계정으로 본다. 어느 프로필과도 맞지 않으면 "프로필에 등록되지 않은 계정"으로 표시한다.

계정 전환은 선택한 프로필의 auth 파일을 Windows 기본 Codex home으로 복사하는 작업이다. 전환은 선택한 프로필 파일과 Windows 기본 파일만 사용해야 하며, 선택하지 않은 프로필 디렉터리의 auth 파일을 수정하면 안 된다.

전환 성공은 Windows 기본 auth가 선택한 프로필과 맞는지 검증된 경우에만 인정한다. 검증 전 복사 명령이 성공했다는 사실만으로는 전환 성공이 아니다.

Codex Desktop 재시작과 SSH Codex 동기화는 전환 이후의 후속 작업이다. Windows 기본 auth 전환이 성공했으면, 재시작이나 SSH가 실패해도 활성 프로필 선택은 유지된다. 대신 후속 작업의 실패 이유와 재시도 가능성을 사용자에게 보여준다.

Logout은 대상 프로필에만 작용한다. 대상 프로필 삭제가 성공하면 프로필 목록, 사용량 목록, tray 상태에서 그 프로필이 사라져야 한다. 삭제된 프로필이 활성 프로필이면 활성 선택은 비워야 하며, 다른 프로필로 몰래 바꾸면 안 된다.

사용량은 프로필별 토큰으로 조회한다. 오류가 난 프로필은 `unknown` 또는 저장된 email과 오류 메시지를 보여줄 수 있지만, 다른 프로필의 사용량 값을 대신 표시하면 안 된다.
