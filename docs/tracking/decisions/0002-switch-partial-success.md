# 전환 결과와 후속 작업 결과 분리

## 상황

사용자는 Windows Codex 계정 전환은 성공했는데 SSH Codex runtime 재시작이 실패하거나 Codex Desktop 실행이 실패하는 상황을 겪었다. 이전 방식처럼 하나의 오류로 처리하면 실제로 어느 부분이 바뀌었는지 알 수 없다.

## 결정

계정 전환 응답을 Windows auth 전환, Desktop 재시작, SSH 동기화로 나눈다. Windows auth 검증이 성공하면 활성 프로필 저장은 유지하고, Desktop 또는 SSH 실패는 별도 경고로 표시한다.

## 대안

하나라도 실패하면 전체 전환을 실패로 볼 수 있다. 이 방식은 단순하지만 Windows auth 파일이 이미 바뀐 뒤 사용자가 실패라고만 보는 문제가 있다.

SSH 실패 시 자동 rollback을 할 수도 있다. 하지만 어떤 이전 프로필로 되돌릴지 명확하지 않고, 또 다른 auth 복사가 일어나 프로필 스냅샷 혼동을 키울 수 있다.

## 결과

사용자는 현재 Windows 계정이 바뀌었는지, Desktop 재시작만 실패했는지, SSH만 실패했는지 구분해서 볼 수 있다. UI는 후속 실패에 재시도를 제공해야 하며, 성공 toast 하나로 모든 단계를 덮으면 안 된다.
