# 0004 사용량 중심 패널

## 결정

React 패널의 첫 화면은 프로필 카드 목록이 아니라 `Codex 사용량` 목록을 중심으로 구성한다.

## 이유

사용자는 계정별 5H/Week 상태와 현재 활성 계정을 가장 먼저 확인해야 한다. 프로필 카드와 사용량 표를 따로 두면 같은 계정 정보와 Login/Run/Logout 액션이 중복되어 전환 실수가 생긴다.

## 결과

- 저장 프로필은 사용량 행에서 전환, 실행, 로그아웃, 숨김, 이름 변경을 처리한다.
- 새 로그인은 프로필 섹션의 icon-only 버튼과 이름 입력 다이얼로그로 시작한다.
- 비활성 프로필 전환은 확인 다이얼로그를 거친다.
- 행의 전환 클릭 영역과 icon action은 DOM에서 분리한다.
